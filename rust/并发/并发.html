说明
定义：并发(Concurrent) 是多个队列使用同一个咖啡机，然后两个队列轮换着使用（未必是 1:1 轮换，也可能是其它轮换规则），最终每个人都能接到咖啡
多核心并发：当核心增多到 N 时，操作系统同时在进行的任务肯定远不止 N 个，这些任务将被放入 M 个线程队列中，接着交给 N 个 CPU 核心去执行，最后实现了 M:N 的处理模型，在这种情况下，并发与并行是同时在发生的，所有用户任务从表面来看都在并发的运行，但实际上，同一时刻只有 N 个任务能被同时并行的处理。
使用线程和通道
use std::thread;
use std::sync::mpsc;

fn main() {
    // 创建一个通道，用于从线程中返回结果
    let (tx, rx) = mpsc::channel();

    // 启动第一个函数的线程
    let tx1 = tx.clone();
    thread::spawn(move || {
        let result = function1();
        tx1.send(result).unwrap();
    });

    // 启动第二个函数的线程
    let tx2 = tx.clone();
    thread::spawn(move || {
        let result = function2();
        tx2.send(result).unwrap();
    });

    // 启动第三个函数的线程
    let tx3 = tx.clone();
    thread::spawn(move || {
        let result = function3();
        tx3.send(result).unwrap();
    });

    // 等待三个通道中的结果，并打印它们
    let result1 = rx.recv().unwrap();
    let result2 = rx.recv().unwrap();
    let result3 = rx.recv().unwrap();

    println!("Result 1: {}", result1);
    println!("Result 2: {}", result2);
    println!("Result 3: {}", result3);
}

fn function1() -> i32 {
    // 第一个函数的实现
    // 返回一个结果
    1
}

fn function2() -> i32 {
    // 第二个函数的实现
    // 返回一个结果
    2
}

fn function3() -> i32 {
    // 第三个函数的实现
    // 返回一个结果
    3
}
