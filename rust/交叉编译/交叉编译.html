mac
目标平台：x86_64 linux
三元组：x86_64-unknown-linux-gnu（动态链接）
这个目标三元组指定了一个使用GNU C库（glibc）的目标系统。GNU C库是一个功能丰富、广泛使用的C标准库，提供了广泛的特性和功能。大多数常见的Linux发行版使用GNU C库作为默认的C标准库。
注：通过gnu的编译方式，glibc需要2.18以上，Centos7默认glibc为2.1.7版本，通过ldd --version查看
添加工具链
rustup target add x86_64-unknown-linux-gnu
基本工具安装
cargo install cargo-zigbuild
brew update-reset
brew upgrade
brew install zig
开始编译
cargo zigbuild --release --target x86_64-unknown-linux-gnu
三元组：x86_64-unknown-linux-musl（静态链接）
编译说明：这个目标三元组指定了一个使用musl库的目标系统。musl库是一个轻量级、高性能的C标准库，专注于简洁性和可移植性。使用这个目标三元组编译的程序将链接到musl库，以在目标系统上运行。musl 牺牲了一些性能和兼容性，换来了完全静态链接的体验，一般情况下都能保证处处皆可运行，而且二进制体积没有很大。
添加工具链
rustup target add x86_64-unknown-linux-musl
编译方式
推荐cargo zigbuild方法
cargo zigbuild
添加工具链
rustup target add x86_64-unknown-linux-musl
cargo install cargo-zigbuild
brew update-reset
brew upgrade
brew install zig
开始编译
cargo zigbuild --release --target=x86_64-unknown-linux-musl
cargo build
musl 在 macOS 上使用 musl-cross, musl-cross 是用来专门编译到 Linux 的工具链， 下面进行安装：
brew install FiloSottile/musl-cross/musl-cross
添加链接
ln -s /usr/local/bin/x86_64-linux-musl-gcc /usr/local/bin/musl-gcc
编译
cargo build --release --target x86_64-unknown-linux-musl
可能的问题
mac端：缺少openssl
  1. 编译安装linux的openssl，参考文档：链接
  2. 编译完成后并打包给mac
3.由mac导出变量，并使得变量生效
export OPENSSL_DIR=/Users/melta/Documents/source/lib/openssl-3.2.1
export OPENSSL_INCLUDE_DIR=$OPENSSL_DIR/include OPENSSL_LIB_DIR=$OPENSSL_DIR/lib64
export X86_64_UNKNOWN_LINUX_MUSL_OPENSSL_INCLUDE_DIR=$OPENSSL_INCLUDE_DIR
  1. 重新编译
linux端：/lib/ld-musl-x86_64.so.1: bad ELF interpreter
sudo yum install https://forensics.cert.org/cert-forensics-tools-release-el7.rpm -y
sudo yum install musl-libc-static -y
linux端：找不到需要的静态文件
OPENSSL=/usr/local/services/openssl-3.2.1/lib64
export LD_LIBRARY_PATH=${OPENSSL}:$LD_LIBRARY_PATH
关于gnu和musl
在 Linux 上，多亏了 musl，静态链接体验相对较好。Linux 上静态链接最主要的坑其实就在于 glibc，如果链接进去可能会存在功能上的问题（参考 Why is statically linking glibc discouraged?），不链接进去的话如果碰上低版本 glibc 的机器程序就无法执行。musl 牺牲了一些性能和兼容性，换来了完全静态链接的体验，一般情况下都能保证处处皆可运行，而且二进制体积没有很大。
