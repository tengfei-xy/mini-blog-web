格式
macro_rules! macro_name {
    // 宏模式
    => {
        // 宏展开内容
    };
}
示例
使用ident关键词的宏定义
ident 是 Rust 中的一个关键字，用于标识符，即变量、函数、模块和宏的名称。
macro_rules! greet {
    ($name:ident) => {
        println!("Hello, {}!", $name);
    };
}
宏使用
greet!(Alice);
宏展开
println!("Hello, Alice!");

todo!()用于标记待办任务。
文档：https://rustwiki.org/zh-CN/std/macro.todo.html
使用说明：当你使用 todo!() 宏时，它会生成一个编译错误，并提醒你该部分代码是一个待办任务，需要进一步处理。
报错场景：在编译、运行时
使用场景：如果你希望确保没有遗漏的待办任务，并在运行时立即发现它们，可以使用 todo!() 宏。
fn main() {
    let result = calculate_something();
    println!("Result: {}", result);
}

fn calculate_something() -> i32 {
    todo!("need coiding") // 待办任务
}
try：解开结果或传播其错误。
unreachable：表示无法访问的代码。
unimplemented!()标记代码中的未实现部分
文档：https://rustwiki.org/zh-CN/std/macro.unimplemented.html
使用说明：当你使用 unimplemented!() 宏时，它会生成一个编译错误，告诉你该部分代码尚未被实现。这通常用于临时占位，以便在后续的开发中进行完善。
报错场景：在编译时
使用场景：如果你希望在开发过程中及时发现未实现的代码并修复它们，可以使用 unimplemented!() 宏。
fn main() {
    let result = calculate_something();
    println!("Result: {}", result);
}

fn calculate_something() -> i32 {
    unimplemented!() // 未实现的代码
}
