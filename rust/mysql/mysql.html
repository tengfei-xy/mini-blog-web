获得连接
get_conn() 方法是一个阻塞方法，这意味着它会一直等待，直到数据库连接池中有一个可用的连接。如果连接池中没有可用的连接，则 get_conn() 方法会返回一个错误。可以多次执行 get_conn() 方法。每次执行 get_conn() 方法都会从数据库连接池中获取一个新的连接。
let mut conn = pool.get_conn().unwrap();

exec方法
conn.prep()
定义：conn.prep() 是 mysql 库中一个用于执行预编译语句的函数。作用：可以提高性能，因为它允许数据库服务器将语句预编译为机器代码。
参数
  • sql: 要执行的预编译语句。
与conn.query()相比
  • conn.prep_exec() 方法可以提高性能，因为它允许数据库服务器将语句预编译为机器代码。
  • conn.prep_exec() 方法可以减少 SQL 注入攻击的风险，因为它允许您将参数与语句分开传递。
示例
use mysql::{params, prelude::Queryable};

    let stmt = conn.prep("INSERT INTO account (username, password) VALUES (:username,:password)");
    web_system_error!(stmt, "注册 数据库预处理失败 {}");

    let result = conn.exec_drop(
        &stmt.unwrap(),
        params! {
            "username" => &req_json.username,
            "password" => new_passwrod,
        },
    );
conn.exec_drop()
使用场景
  • 执行不需要返回值的 SQL 语句，例如 INSERT、UPDATE、DELETE 等。
  • 需要提高性能的场景，例如批量插入数据。
  • 需要减少内存使用量，例如在处理大型数据集时。
示例
包含参数

不包含参数

在actix_web中使用mysql
说明：在 Rust 中，可以使用 mysql 库来连接 MySQL 数据库。在 Actix Web 服务器中，可以使用 Data 结构来存储数据库连接池，并将其注入到任何请求处理程序中。以下是使用 mysql 和 Actix Web 进行数据库查询的步骤：
添加依赖项
cargo add mysql
创建数据库连接池
use mysql::Pool;

fn init_mysql() -> Pool {
    let username = "";
    let password = "";
    let host = "";
    let database_name = "";   
    Pool::new(format!("mysql://{}:{}@{}/{}",username,password,host,database_name).as_str()).unwrap()
}
创建 Data 结构
use actix_web::web::Data;

struct MysqlData {
    pool: Data<Pool>,
}
将数据库连接池注入到请求处理程序中
use actix_web::{get, web, App, HttpServer, web:Data};

#[get("/")]
async fn index(data: Data<MysqlData>) -> impl Responder {
    let mut conn = data.pool.get_conn().unwrap();
    let results: std::prelude::v1::Result<Vec<Row>, Error> =
        conn.query("SELECT username from account");

    // ...

    Ok("Hello, world!")
}

fn main() {

     HttpServer::new(|| {
        App::new()
            .app_data(Data::new(init_mysql().clone()))
            .route("/api/v1/login", web::post().to(login))
            .route("/api/v1/sigup", web::post().to(sigup))
    })
    .bind("localhost:8080")
    .unwrap()
    .run()
    .unwrap();
}
在上述代码中，我们首先添加了 mysql 和 actix-web 依赖项。然后，我们创建了一个 Pool 对象来存储数据库连接池。接下来，我们创建了一个 Data 结构来存储数据库连接池。最后，我们将数据库连接池注入到请求处理程序中。
在请求处理程序中，我们可以使用 get_conn() 方法从连接池中获取一个连接。然后，我们可以使用 query() 方法执行数据库查询。
