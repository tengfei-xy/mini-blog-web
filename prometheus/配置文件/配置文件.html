global:
  scrape_interval:     15s
  evaluation_interval: 15s

rule_files:
  # - "first.rules"
  # - "second.rules"

scrape_configs:
  - job_name: prometheus
    static_configs:
      - targets: ['localhost:9090']
参数说明
global 块
作用：控制 Prometheus 服务器的全局配置。
scrape_interval
作用：控制 Prometheus 抓取目标的频率
evaluation_interval
作用：控制 Prometheus 评估规则的频率。Prometheus 使用规则创建新的时间序列并生成警报。
rule_files
作用：指定了我们希望 Prometheus 服务器加载的任何规则的位置。目前，我们没有规则。
scrape_configs
作用：控制 Prometheus 监视的资源。
说明：由于 Prometheus 还将有关自身的数据公开为 HTTP 端点，因此它可以抓取和监控自己的运行状况。
默认配置：有一个名为 prometheus 的作业，用于抓取 Prometheus 服务器公开的时间序列数据。该作业包含一个静态配置的目标，即 localhost on 端口 9090 。Prometheus 期望指标在路径为 /metrics .因此，此默认作业是通过 URL： http://localhost:9090/metrics 进行抓取。
job_name
用于标识一个数据采集任务的名称。它可以用来在 Prometheus 中区分不同的数据采集源，也可以用来在规则中过滤数据。
示例
日志选项
修改日志等级，支持debug, info, warn, error
./prometheus --config.file=prometheus.yml --log.level=debug
修改日志格式，支持logfmt,json
./prometheus --config.file=prometheus.yml --log.format=logfmt
