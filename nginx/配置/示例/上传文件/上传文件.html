修改上传文件大小
client_max_body_size：这个配置项用于限制客户端请求的最大请求体大小，包括文件上传。默认情况下，其值为1m（1兆字节）。您可以通过将其设置为更大的值来增加最大文件上传大小。例如，要将最大文件上传大小设置为100兆字节，您可以将以下配置添加到NGINX的配置文件中：
client_body_buffer_size：这个配置项用于设置客户端请求体缓冲区的大小。默认情况下，其值与client_max_body_size相同。如果您在处理大文件上传时遇到内存不足的问题，可以尝试增加这个值。例如，要将客户端请求体缓冲区的大小设置为256兆字节，您可以将以下配置添加到NGINX的配置文件中：
http {
    client_max_body_size 100m;
    client_body_buffer_size 256m;
}
