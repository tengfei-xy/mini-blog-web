作用：提供了一种灵活且高效的方式来描述、查询和操作数据，为客户端应用程序提供了更多的控制权
核心思想：将数据的获取和传输的过程交给客户端来定义。
传输过程：客户端可以通过编写一个包含所需数据字段和关联关系的查询来告诉服务器需要什么数据。服务器会根据查询的结构和参数返回与之匹配的数据，确保客户端只获取到所需的数据，避免了服务器返回过多或过少的数据的问题。
特点
强大的类型系统：GraphQL 使用类型来定义数据模型，客户端和服务器可以共享这些类型定义，确保数据的一致性和完整性。
单一请求：GraphQL 允许客户端在单个请求中获取多个资源的数据，减少了网络请求的次数，提高了性能。
关联查询：GraphQL 支持在一个查询中获取相关联的数据，避免了传统 RESTful API 中多次请求关联数据的问题。
可编写的查询：GraphQL 查询是可编写的，客户端可以根据需要自由组织和嵌套查询，获取所需的数据，提供了更大的灵活性。
和RESTful API相比
GraphQL 允许客户端精确地指定需要从服务器获取的数据，避免了过度获取和多次请求的问题。
