使用方法
查看内存使用情况
jmap -heap pid
查看内存对象数量和大小
# 注：JVM会先触发gc，然后再统计信息。
jmap -histo:live pid
将内存使用情况dump到文件中
# 注： 这个命令执行，JVM会将整个heap的信息dump写入到一个文件，heap如果比较大的话，就会导致这个过程比较耗时
# 注： 执行的过程中为了保证dump的信息是可靠的，所以会暂停应用。
jmap -dump:live,format=b,file=filename pid
统计perm区
# 注：这整个过程也会比较的耗时，并且同样也会暂停应用。
jmap -permstat
影响
方式
会暂停程序执行
jmap命令会发送一个请求到JVM，要求JVM暂停程序执行，并获取内存使用情况。因此，在jmap命令执行期间，程序将无法继续执行。
会占用额外的内存
jmap命令会在JVM的内存中创建一个内存映射文件，用于存储内存使用情况。因此，在jmap命令执行期间，JVM将会使用更多的内存。
可能会导致性能下降
jmap命令会对JVM进行额外的操作，因此可能会导致程序的性能下降。
因素
程序的运行状态
如果程序处于空闲状态，则jmap命令的影响较小。
如果程序处于繁忙状态，则jmap命令的影响较大。
程序的内存使用情况
如果程序的内存使用情况较小，则jmap命令的影响较小。
如果程序的内存使用情况较大，则jmap命令的影响较大。
使用建议
  • 尽量在程序处于空闲状态时使用jmap命令。
  • 如果程序的内存使用情况较大，则应尽量减少jmap命令的执行时间。
  • 在程序启动后，先让程序运行一段时间，以便程序的内存使用情况趋于稳定后再使用jmap命令。
  • 使用jmap命令时，指定-dump参数，将内存使用情况保存到文件中。这样，可以避免在jmap命令执行期间暂停程序执行。
  • 使用jmap命令时，指定-histo参数，可以生成内存使用情况的直方图。这样，可以快速查看内存使用情况的概况。
