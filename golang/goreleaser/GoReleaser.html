快速开始：参考
发布
发行
  1. 检查配置文件
    goreleaser check
  2. 设定环境变量GITHUB_EXPORT并导出，token在这里创建，最小需要write:packages权限
  3. 在项目更新完毕之后，设置tag
    git tag -a v0.1.0 -m "First release"
git push origin v0.1.0
  4. 不正式发release包，执行下流程，看看有没有一些执行错误 
    goreleaser release --snapshot --clean
  5. 创建发布
    goreleaser release
测试
本地发布包测试 
goreleaser build --single-target
配置
增加文件到打包结果中
archives:
  - format: tar.gz
    files:
      - sql/
      - config.yaml.save
