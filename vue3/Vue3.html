使用Vue方式
全局构建
<script src="https://unpkg.com/vue@3/dist/vue.global.js"></script>

<div id="app">{{ message }}</div>

<script>
  const { createApp } = Vue
  
  createApp({
    data() {
      return {
        message: 'Hello Vue!'
      }
    }
  }).mount('#app')
</script>
ES模块
使用了 <script type="module">，且导入的 CDN URL 指向的是 Vue 的 ES 模块构建版本。
<div id="app">{{ message }}</div>

<script type="module">
  import { createApp } from 'https://unpkg.com/vue@3/dist/vue.esm-browser.js'
  
  createApp({
    data() {
      return {
        message: 'Hello Vue!'
      }
    }
  }).mount('#app')
</script>
导入映射表（Import Maps）
<script type="importmap">
  {
    "imports": {

    }
  }

API FAQ
选项式API
逻辑复用机制：mixins
作用：用包含多个选项（data、methods和 mounted）的对象来描述组件的逻辑
效果：定义的属性会暴露在函数内部的this上，他会指向当前的组件示例
适合场景：中小项目
组合式API
<script setup> 是SFC 与组合式 API 时该语法是默认推荐
定义：组合式 API (Composition API) 是一系列 API 的集合，
意义：使得可以使用函数而不是声明选项的方式书写 Vue 组件
优势：实现更加简洁高效的逻辑复用，解决了Mixins的所有缺陷
用法：使用导入的 API 函数来描述组件逻辑
适合场景：大型项目
包含内容
响应式 API：例如 ref() 和 reactive()，使我们可以直接创建响应式状态、计算属性和侦听器。
生命周期钩子：onMounted() 和 onUnmounted()，使我们可以在组件各个生命周期阶段添加逻辑。
依赖注入：provide() 和 inject()，使我们可以在使用响应式 API 时，利用 Vue 的依赖注入系统。
Vue应用
创建应用——createApp()
说明：每个Vue应用都是通过此函数创建Vue应用示例，同时拥有自己的配置和全局资源作用域，并返回根组件。
根组件
定义：根组件是指应用程序的启动点，是 Vue 实例。
作用：负责初始化应用程序并挂载到 DOM 元素上。
template
设置：Vue会自动使用根组件挂载元素的innerHTML作为根模板
未设置：Vue会使用指定的模板作为根模板
根模板
定义：根模板是指应用的入口点，它定义了应用的初始结构和内容。根模板可以是 HTML 代码、JavaScript 代码或两者混合。
来源：根组件模板是组件本身的一部分，或在挂载容器内编写模板来单独提供。
挂载应用——.mount()
容器元素自己将不会被视为应用的一部分。
说明：应用实例创建后，必须调用此方法后应用根组件的内容才会渲染出来。
方法
1. 此方法应该始终在整个应用配置和资源注册完成后被调用。 2. 返回值是根组件实例而非应用实例。
说明：此方法接收一个“容器”参数，可以是实际的DOM元素或CSS选择器字符串
配置应用——.config
说明：应用实例会暴露一个 .config 对象允许我们配置一些应用级的选项
示例
定义一个应用级的错误处理器，用来捕获所有子组件上的错误：
app.config.errorHandler = (err) => {
  /* 处理错误 */
}
注册应用范围内可用的资源，这使得 TodoDeleteButton 在应用的任何地方都是可用的。
app.component('TodoDeleteButton', TodoDeleteButton)
