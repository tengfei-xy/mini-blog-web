本文档基于TypeScript
目的：只用一个html文件，完成多个站内页面渲染、跳转的功能
相关页面：在组件内使用路由
目录结构
src
│ # 路由目录
├─router
│   # 路由入口文件
├───index.ts
│   # 路由配置，如果路由很多，可以再拆分模块文件
├───routes.ts
│ # 项目入口文件
└─main.ts
routes.ts：成为路由树，可以添加多级路由
复杂的项目：拆分成两个文件 index.ts 和 routes.ts ，在 routes.ts 里维护路由树的结构，在 index.ts 导入路由树结构并激活路由，同时可以在该文件里配置路由钩子。
更复杂的项目：例如做一个 Admin 后台，可以按照业务模块，再把 routes 拆分得更细，例如 game.ts / member.ts / order.ts 等业务模块，再统一导入到 index.ts 文件里。
引入路由
import { createRouter, createWebHistory } from 'vue-router'
import type { RouteRecordRaw } from 'vue-router'

const routes: Array<RouteRecordRaw> = [
  // ...
]

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes,
})

export default router
路由渲染
所有路由组件，要在访问后进行渲染，都必须在父级组件里带有 <router-view /> 标签。
说明：<router-view /> 在哪里，路由组件的代码就渲染在哪个节点上，一级路由的父级组件，就是 src/App.vue 这个根组件。
总页面
<template>
  <router-view />
</template>
总分总页面
页首页尾不变，中间区域是路由
<template>
  <!-- 全局页头 -->
  <Header />

  <!-- 路由 -->
  <router-view />

  <!-- 全局页脚 -->
  <Footer />
</template>
部分页面
大部分页面都需要有侧边栏，但登录页、注册页不需要。
可以通过 路由元信息 来管理这些规则。
<template>
  <!-- 登录 -->
  <Login v-if="route.name === 'login'" />

  <!-- 注册 -->
  <Register v-else-if="route.name === 'register'" />

  <!-- 带有侧边栏的其他路由 -->
  <div v-else>
    <!-- 固定在左侧的侧边栏 -->
    <Sidebar />

    <!-- 路由 -->
    <router-view />
  </div>
</template>

文件名：routes.ts
组成部分：类型声明、数组结构、模块导出

const routes: Array<RouteRecordRaw> = [
  // ...
]

配置方式
n级路由
一级路由
一级路由的 path 都必须是以 / 开头，比如： /home、/setting
path ：路由的访问路径
name：路由的名称，非必填。代替path实现跳转
一级路由：https://example.com/home 这里的 home 就是一级路由。
component ：路由的模板文件，指向一个 vue 组件，用于指定路由在浏览器端的视图渲染，包含两种组件加载方式
同步组件
// src/router/routes.ts

// 使用 TypeScript 时需要导入路由项目的类型声明
import type { RouteRecordRaw } from 'vue-router'

// 一级路由
const routes: Array<RouteRecordRaw> = [
  {
    path: '/',
    name: 'home',
    component: () => import('@views/home.vue'),
  },
]

// 将路由数组导出给其他模块使用
export default routes
异步组件（路由懒加载）
// src/router/routes.ts

// 使用 TypeScript 时需要导入路由项目的类型声明
import type { RouteRecordRaw } from 'vue-router'
import Home from '@views/home.vue'

// 一级路由
const routes: Array<RouteRecordRaw> = [
  {
    path: '/',
    name: 'home',
    component: Home,
  },
]

// 将路由数组导出给其他模块使用
export default routes
多级路由
网址：https://example.com/chinese-food/dumplings
一级路由：chinese-food
二级路由：dumplings
示例
const routes: Array<RouteRecordRaw> = [
  // 注意：这里是一级路由
  {
    path: '/lv1',
    name: 'lv1',
    component: () => import('@views/lv1.vue'),
    // 注意：这里是二级路由，在 `path` 的前面没有 `/`
    children: [
      {
        path: 'lv2',
        name: 'lv2',
        component: () => import('@views/lv2.vue'),
        // 注意：这里是三级路由，在 `path` 的前面没有 `/`
        children: [
          {
            path: 'lv3',
            name: 'lv3',
            component: () => import('@views/lv3.vue'),
          },
        ],
      },
    ],
  },
]

https://example.com/lv1/lv2/lv3
路由功能
重定向 / redirect
只能针对那些不带参数的路由
使用原因：对一些已下线的页面，直接访问原来的地址会导致 404 ，为了避免这种情况出现，通常会配置重定向将其指向一个新的页面，或者跳转回首页。
参数
类型值
string另外一个路由的 path
route另外一个路由（类似 router.push）
function可以判断不同情况的重定向目标，最终 return 一个 path 或者 route
用法
路径参数
不带
一般用法
const routes: Array<RouteRecordRaw> = [
  {
    path: '/',
    name: 'home',
    component: () => import('@views/home.vue'),
    meta: {
      title: 'Home',
    },
  },
  // 访问这个路由会被重定向到首页
  {
    path: '/error',
    redirect: '/',
  },
]
指向首页
const routes: Array<RouteRecordRaw> = [
  // 重定向到 `/home`
  {
    path: '/',
    redirect: '/home',
  },
  // 真正的首页
  {
    path: '/home',
    name: 'home',
    component: () => import('@views/home.vue'),
  },
]
带
最终访问的地址：https://example.com/home?from=redirect
const routes: Array<RouteRecordRaw> = [
  // 重定向到 `/home` ，并带上一个 `query` 参数
  {
    path: '/',
    redirect: {
      name: 'home',
      query: {
        from: 'redirect',
      },
    },
  },
  // 真正的首页
  {
    path: '/home',
    name: 'home',
    component: () => import('@views/home.vue'),
  },
]
根据变量指定路径
const routes: Array<RouteRecordRaw> = [
  // 访问主域名时，根据用户的登录信息，重定向到不同的页面
  {
    path: '/',
    redirect: () => {
      // `loginInfo` 是当前用户的登录信息
      // 可以从 `localStorage` 或者 `Pinia` 读取
      const { groupId } = loginInfo

      // 根据组别 ID 进行跳转
      switch (groupId) {
        // 管理员跳去仪表盘
        case 1:
          return '/dashboard'

        // 普通用户跳去首页
        case 2:
          return '/home'

        // 其他都认为未登录，跳去登录页
        default:
          return '/login'
      }
    },
  },
]
路由别名 / alias
实现可以通过 /home 访问首页，也可以通过 /index 访问首页
const routes: Array<RouteRecordRaw> = [
  {
    path: '/home',
    alias: '/index',
    name: 'home',
    component: () => import('@views/home.vue'),
  },
]
路径匹配
404
新版的路由不再支持直接配置通配符 * ，而是必须使用带有自定义正则表达式的参数进行定义，详见官网 删除了 *（星标或通配符）路由 的说明。
const routes: Array<RouteRecordRaw> = [
  {
    path: '/:pathMatch(.*)*',
    name: '404',
    component: () => import('@views/404.vue'),
  },
]
钩子
作用：在所有的路由在被访问到的时候，都会触发这些钩子函数。
参数示例
beforeEach
名称：全局前置守卫
触发时机：在路由跳转前触发
参数
to：即将要进入的路由对象
from：当前导航正要离开的路由
示例
设定路由的网页标题
router.beforeEach((to, from) => {
  const { title } = to.meta
  document.title = title || '默认标题'
})
设定是否需要登录
router.beforeEach((to, from) => {
  const { isNoLogin } = to.meta
  if (!isNoLogin) return '/login'
})
参数丢失时，返回首页
router.beforeEach((to) => {
  if (to.name === 'article' && to.matched.length === 0) {
    return '/'
  }
})
beforeResolve
名称：全局解析守卫
触发时机：在导航被确认前，同时在组件内守卫和异步路由组件被解析后
使用场景：它通常会用在一些申请权限的环节，比如一些 H5 页面需要申请系统相机权限、一些微信活动需要申请微信的登录信息授权，获得权限之后才允许获取接口数据和给用户更多的操作。
触发顺序：beforeEach（全局） > beforeEnter（独享） > beforeResolve（全局）。
参数
to：即将要进入的路由对象
from：当前导航正要离开的路由
afterEach
名称：全局后置守卫
触发时机：在路由跳转完成后触发
使用场景：统计
参数
to：即将要进入的路由对象
from：当前导航正要离开的路由
示例
全局配置
在 src/router/index.ts 里，在创建路由之后、在导出去之前使用：
import { createRouter } from 'vue-router'

// 创建路由
const router = createRouter({ ... })

// 在这里调用导航守卫的钩子函数
router.beforeEach((to, from) => {
  // ...
})

// 导出去
export default router
元信息用法
自定义信息
字段类型含义
titlestring用于在渲染的时候配置浏览器标题；
isDisableBreadcrumbLinkboolean是否禁用面包屑链接（对一些没有内容的路由可以屏蔽访问）；
isShowBreadcrumbboolean是否显示面包屑（此处的登录页不需要面包屑）；
addToSidebarboolean是否加入侧边栏（此处的登录页不需要加入侧边栏）；
sidebarIconstring配置侧边栏的图标 className（默认）；
sidebarIconAltstring配置侧边栏的图标 className（展开状态）；
isNoLoginboolean是否免登录（后台默认强制登录，设置为 true则可以免登录访问，此处的登录页不需要校验）；
const routes: Array<RouteRecordRaw> = [

    path: '/login',
    name: 'login',
    component: () => import('@views/login.vue'),

      title: '登录',
      isDisableBreadcrumbLink: true,
      isShowBreadcrumb: false,
      addToSidebar: false,
      sidebarIcon: '',
      sidebarIconAlt: '',
      isNoLogin: true,
    },
  },
]

自定义标签
通过使用：custom和v-slot渲染为其他标签
属性说明
custom：一个布尔值，用于控制是否需要渲染为 a 标签，当不包含 custom 或者把 custom 设置为 false 时，则依然使用 a 标签渲染。
v-slot
v-slot 是一个对象，用来决定标签的行为，它包含了
字段含义
href解析后的 URL，将会作为一个 a 元素的 href 属性
route解析后的规范化的地址
navigate触发导航的函数，会在必要时自动阻止事件，和 router-link 同理
isActive如果需要应用激活的 class 则为 true，允许应用一个任意的 class
isExactActive如果需要应用精确激活的 class 则为 true，允许应用一个任意的 class
<template>
  <router-link to="/home" custom v-slot="{ navigate }">
    <span class="link" @click="navigate"> 首页 </span>
  </router-link>
</template>
