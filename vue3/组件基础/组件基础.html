定义组件
名称格式
在DOM 内模板不可用
格式：使用 PascalCase 作为组件名的注册格式
说明：以 MyComponent 为名注册的组件，在模板中可以通过 <MyComponent> 或 <my-component> 引用。这让我们能够使用同样的 JavaScript 组件注册代码来配合不同来源的模板。
缘由
  1. PascalCase 是合法的 JavaScript 标识符。这使得在 JavaScript 中导入和注册组件都很容易，同时 IDE 也能提供较好的自动补全。
  2. <PascalCase /> 在模板中更明显地表明了这是一个 Vue 组件，而不是原生 HTML 元素。同时也能够将 Vue 组件和自定义元素 (web components) 区分开来。

ts / <script lang="ts">/ defineCompnent()
定义：是 Vue 3 推出的一个全新 API
作用：可用于对 TypeScript 代码的类型推导，帮助开发者简化掉很多编码过程中的类型声明。
说明：为了让 TypeScript 正确地推导出组件选项内的类型，我们需要通过 defineComponent() 这个全局 API 来定义组件，使用 defineComponent() 能够包裹组件的内部逻辑
数据导出说明：需要在 <template /> 中使用，就必须在 setup 里将其 return 出去
示例
<script lang="ts">

import { defineComponent } from 'vue'

export default defineComponent({
  // 启用了类型推导
  props: {
    name: String,
    msg: { type: String, required: true }
  },
  data() {
    return {
      count: 1
    }
  },
  mounted() {
    this.name // 类型：string | undefined
    this.msg // 类型：string
    this.count // 类型：number
  }
})
</script>

<template>
  <header>

    <div class="wrapper">
      <p>
        <button @click="count++">count is: {{ count }}</button>
      </p>
      <!-- <HelloWorld msg="You did it!" /> -->
    </div>
  </header>
</template>

组合式API
js
<scipt>
不使用构建步骤
组件选项 / setup()
使用此<script setup> ，不可以在中export default 里配置setup()，二者选其一
setup 函数中的代码在组件实例创建之前执行，因此无法使用this访问组件实例。 如果需要访问组件实例，可以使用 context 对象的 attrs 属性，或者在 setup 函数内部创建一个响应式的引用，并将其赋值为 this。
定义：是Vue3的组件选项，也是使用组合式 API 的入口
使用场景
  1. 需要在非单文件组件中使用组合式 API 时。
  2. 需要在基于选项式 API 的组件中集成基于组合式 API 的代码时。
作用：用于配置组件的初始状态、引入组件所需的响应式数据、注册事件处理函数等。
目的：是为组件提供一种更直接的方式来声明和使用组件的属性、方法和生命周期钩子。
执行顺序：在创建组件之前执行,在 setup 执行之后，其他的生命周期才会被启用
接收参数
props
定义：是一个由父组件传递下来的object响应式对象
是否必传：否
说明：当父组件传入新的数据时，它将被更新。
使用方式：通过解构（会失去响应性）或直接使用该对象来访问和使用这些属性。
context
定义：是一个组件的执行上下文object对象
是否必传：否
是否支持结构：是
内容（暴露了组件的属性）
attrs
类型：非响应式对象
作用：未在 Props 里定义的属性都将变成 Attrs
使用方式：attrs.xxx
slots
类型：非响应式对象
作用：组件插槽，用于接收父组件传递进来的模板内容
使用方式：slots.xxx
emit
类型：方法
作用：触发父组件绑定下来的事件
使用方式：{emit}
平时使用可以通过直接传入 { emit } ，即可用 emit('xxx') 来代替使用 context.emit('xxx')，另外两个功能也是如此。

返回参数
内容：响应式数据、计算属性、方法等。
说明：返回的内容成为组件示例的一部分
说明：在一个 .js 文件里默认导出了它自己。
模板：这里的模板是一个内联的 JavaScript 字符串，Vue 将会在运行时编译它。
import { ref } from 'vue'

export default {
  setup() {
    const count = ref(0)
    return { count }
  },
  template: `
    <button @click="count++">
      You clicked me {{ count }} times.
    </button>`
  // 也可以针对一个 DOM 内联模板：
  // template: '#my-template-element'
}
<script setup>
使用此<script setup> ，不可以在中export default 里配置setup()，二者选其一
说明：<script setup> 是在单文件组件 (SFC) 中使用组合式 API 的编译时语法糖。当同时使用 SFC 与组合式 API 时该语法是默认推荐。
优势
  • 更少的样板内容，更简洁的代码。
  • 能够使用纯 TypeScript 声明 props 和自定义事件。
  • 更好的运行时性能 (其模板会被编译成同一作用域内的渲染函数，避免了渲染上下文代理对象)。
  • 更好的 IDE 类型推导性能 (减少了语言服务器从代码中抽取类型的工作)。
基本语法
下列代码会被编译成组件 setup() 函数的内容。这意味着与普通的 <script> 只在组件被首次引入的时候执行一次不同，<script setup> 中的代码会在每次组件实例被创建的时候执行。
<script setup>
console.log('hello script setup')
</script>
示例
<script setup>
import { ref, onMounted } from 'vue'

// 响应式状态
const count = ref(0)

// 用来修改状态、触发更新的函数
function increment() {
  count.value++
}

// 生命周期钩子
onMounted(() => {
  console.log(`The initial count is ${count.value}.`)
})
</script>

<template>
  <button @click="increment">Count is: {{ count }}</button>
</template>

import { defineComponent } from 'vue'

// 使用 `defineComponent` 包裹组件代码
// 即可获得完善的 TypeScript 类型推导支持
export default defineComponent({
  setup() {
    // 在 `setup` 方法里声明变量

有响应式数据
<template>
  <p>{{ msg }}</p>

  <!-- 在这里点击执行 `return` 出来的方法 -->
  <button @click="updateMsg">修改MSG</button>
</template>

<script lang="ts">
import { defineComponent, onMounted, ref } from 'vue'

export default defineComponent({
  setup() {
    const msg = ref<string>('Hello World!')

    // 这个要暴露给模板使用，必须 `return` 才可以使用
    function updateMsg() {
      msg.value = 'Hi World!'
    }

    // 这个要在页面载入时执行，无需 `return` 出去
    const init = () => {
      console.log('init')
    }

    onMounted(() => {
      init()
    })

    return {
      msg,
      updateMsg,
    }
  },
})
</script>
注册组件
一个 Vue 组件在使用前需要先被“注册”，这样 Vue 才能在渲染模板时找到其对应的实现。
流程
1、main.js 中使用 app.use(library)安装组件库
2、app.use 会调用 library 导出的 install 函数
3、在 install 函数内使用 app.component 注册组件
4、然后组件就注册完毕,可以在其他文件使用 import 引入了
说明：
  • install 函数负责组件的注册工作
  • import 只是导入组件定义,但并没有注册
  • 实际上是 Vue.use -> 调用 install -> 在 install 内注册组件
  • 这样组件才真正安装和注册完毕,才可以在其他文件使用 所以 install 函数虽然不直接在 import 时调用,但它负责组件注册这一关键步骤,使得 import 后组件可以正常使用。
方式
局部注册
局部注册的组件在后代组件中并不可用,仅能在父组件中使用
优点：使组件之间的依赖关系更加明确，并且对 tree-shaking 更加友好。
单文件组件
无需注册
<script setup>
import ComponentA from './ComponentA.vue'
</script>

<template>
  <ComponentA />
</template>
不使用构建步骤
说明：需要使用 components 选项来显式注册
import ComponentA from './ComponentA.js'

export default {
  components: {
    // 写法一
    ComponentA
    // 写法二
    ComponentA: ComponentA
  },
  setup() {
    // ...
  }
}
全局注册
使用方法：.component()
使用说明：支持链式调用（app.component('A'.A).component('B'.B).component('C'.C)）
子组件：所有的子组件也可以使用全局注册的组件，这意味着这三个组件也都可以在彼此内部使用。
缺点
  1. 如果全局注册了一个组件，但没有实际使用，它仍然会出现在打包后的 JS 文件中。即无法自动移 (也叫“tree-shaking”)
  2. 全局注册在大型项目中使项目的依赖关系变得不那么明确。在父组件中使用子组件时，不太容易定位子组件的实现。和使用过多的全局变量一样，这可能会影响应用长期的可维护性。
使用单文件组件
import MyComponent from './App.vue'

app.component('MyComponent', MyComponent)
不使用构建步骤
import { createApp } from 'vue'

const app = createApp({})

app.component(
  // 注册的名字
  'MyComponent',
  // 组件的实现
  {
    /* ... */
  }
)
使用子组件
使用一个子组件，并在父组件中导入它,通过 <script setup>，导入的组件都在模板中直接可用。
<script setup>
import ButtonCounter from './ButtonCounter.vue'
</script>

<template>
  <h1>Here is a child component!</h1>
  <ButtonCounter />
</template>
