请注意设置的区域，默认是public，有时是trusted
初始化
开启IP伪装功能
sudo firewall-cmd --add-masquerade --permanent
sudo firewall-cmd --reload
查询IP伪装功能
sudo firewall-cmd --query-masquerade

# 移除伪装
sudo firewall-cmd --remove-masquerade
此时
[root@localhost ~]# cat /proc/sys/net/ipv4/ip_forward
1
[root@localhost ~]# cat /proc/sys/net/ipv4/conf/eth0
1
开启本地路由，其中eth0是机器的网卡名，注意替换
# sysctl -a 2>1 | grep ipv4\.conf | grep route_localnet
sudo sysctl -w net.ipv4.conf.eth0.route_localnet=1


添加转发规则
toport= 和toaddr= 至少要指定一个 由--permanent指定的命令，需要重启(firewall-cmd --reload）才能生效
语法规则
forward-port port=<PORTNUM> protocol=tcp|udp [to-port=<PORTNUM> ] [to-addr=<ADDRESS> ]
示例
任何来自:443 转发至 20.115.226.155:443
firewall-cmd --permanent --add-forward-port=port=443:proto=tcp:toport=443:toaddr=170.106.183.135
任何来自:9002 转发至 127.0.0.1:9001
firewall-cmd --permanent --add-forward-port=port=9002:proto=tcp:toport=9001:toaddr=127.0.0.1
转发从192.168.0.0/24来的，发往80/TCP的流量到防火墙的端口8080/TCP
firewall-cmd --add-rich-rule='rule family=ipv4 source address=192.168.0.0/24 forward-port port=80 protocol=tcp to-port=8080'
查找转发规则
# 详写
firewall-cmd --list-forward-ports
# 简写
firewall-cmd --list-for
删除转发规则（建议先查找，然后直接复制）
firewall-cmd --permanent --remove-forward-port=port=28019:proto=tcp:toport=28018:toaddr=20.115.226.155

# 需要重启
firewall-cmd --reload
