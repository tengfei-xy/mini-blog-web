说明
Permissive 模式
宽容模式，在 Permissive 模式下，SELinux 会正常加载并运行，但不会限制任何操作。如果一个操作不符合 SELinux 的安全策略，SELinux 会发出警告，但不会阻止该操作的执行。
Permissive 模式适合在测试或开发环境中使用，可以帮助用户了解 SELinux 的安全策略，并找出可能存在的安全漏洞。
Enforcing 模式
强制模式，在 Enforcing 模式下，SELinux 会严格执行安全策略。如果一个操作不符合 SELinux 的安全策略，SELinux 会拒绝该操作的执行。
Enforcing 模式是 SELinux 的默认模式，适合在生产环境中使用。
Disabled 模式
关闭模式，在 Disabled 模式下，SELinux 的政策文件不会被加载，SELinux 不会执行任何安全检查。
Disabled 模式适合在需要关闭 SELinux 的环境中使用，例如在使用第三方安全软件时。
查看SELinux状态
# 方法一，如果SELinux status参数为enabled即为开启状态
/usr/sbin/sestatus -v
# 方法二
getenforce
关闭SELinux
临时关闭
setenforce 0
永久关闭（重启生效）
# 将SELINUX=enforcing改为SELINUX=disabled
vi /etc/selinux/config
快速关闭
sudo sed -i 's/SELINUX=enforcing/SELINUX=disabled/' /etc/selinux/config && sduo setenforce 0
开启SELinux
设置SELinux 成为Enforcing模式
setenforce 1
