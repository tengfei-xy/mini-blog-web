Type
simple
  • 这是最基本的服务类型。
  • 服务在系统启动时启动，并在系统关闭时停止。
  • 服务不会被重新启动，即使它崩溃或退出。
forking
  • 这种服务类型与 simple 类型类似，但它会在后台运行一个子进程。
  • 子进程负责提供服务，而父进程负责监视子进程并确保它继续运行。
  • 如果子进程崩溃或退出，父进程会自动重新启动它。
oneshot
  • 这种服务类型只运行一次，然后退出。
  • 它通常用于执行一次性任务，例如安装软件或更新系统。
dbus
  • 这种服务类型使用 D-Bus 消息总线与其他服务通信。
  • D-Bus 是一个进程间通信系统，它允许服务相互发送消息。
  • 这使服务能够相互协调并交换信息。
notify
  • 这种服务类型在启动和停止时发送通知。
  • 通知可以被其他服务或脚本捕获，以便它们可以对服务的状态做出反应。
notify 是一种特殊的 systemd 服务类型，它允许服务在启动和停止时发送通知。通知可以被其他服务或脚本捕获，以便它们可以对服务的状态做出反应。
notify 服务的配置文件与其他类型服务的配置文件类似，但它需要在 [Service] 段落中添加 NotifyAccess=all 选项。这将允许服务向所有用户发送通知。
[Unit]
Description=My Notify Service

[Service]
Type=notify
NotifyAccess=all
ExecStart=/usr/bin/my_service_start
ExecStop=/usr/bin/my_service_stop

[Install]
WantedBy=multi-user.target
当 notify 服务启动或停止时，它会发送一个 D-Bus 消息。消息的格式如下：
org.freedesktop.systemd1.start-stop-finish
消息中包含以下信息：
  • 服务的名称
  • 服务的状态（已启动或已停止）
  • 服务的退出代码（如果服务已停止）
其他服务或脚本可以通过监听 D-Bus 消息总线来捕获 notify 服务发送的通知。当收到通知时，服务或脚本可以执行相应的操作，例如记录服务的状态、重新启动服务或通知用户服务的状态。
以下是一个示例脚本，它监听 D-Bus 消息总线并打印出 notify 服务发送的通知：
#!/bin/bash

# 监听 D-Bus 消息总线
dbus-monitor --session "interface='org.freedesktop.systemd1'" | while read line; do

    # 解析 D-Bus 消息
    service=$(echo $line | grep -oP '(?<=Unit )(.*)(?= )')
    status=$(echo $line | grep -oP '(?<=Status )(.*)(?= )')
    exit_code=$(echo $line | grep -oP '(?<=Result )(.*)(?= )')

    # 打印出通知信息
    echo "Service: $service"
    echo "Status: $status"
    echo "Exit code: $exit_code"

done
你可以将这个脚本保存为一个文件，例如 notify_listener.sh，然后使用以下命令运行它：
./notify_listener.sh
现在，当任何 notify 服务启动或停止时，你都会在终端中看到一条通知消息。
idle
  • 这种服务类型只在系统空闲时运行。
  • 这通常用于执行不需要大量资源的任务，例如备份或索引文件。
timer
  • 这种服务类型在指定的时间间隔运行。
  • 这通常用于执行定期任务，例如发送电子邮件或更新软件。
